Accuracy Robot (Apr. 2024 - Aug.2024)

As part of my summer 2024 internship at MOLLI Surgical, which was adopted by Stryker, I worked on developing an accuracy testing robot named Roberto. The purpose of Roberto was to take magnetic field measurements at commanded locations, which made a shape, and output them to a CSV file. This was then used to enhance the precision of the company's product. I created the software as well as refined models of the wand connector and magnet connector using Fusion 360. The robot could then be used to test other wand errors by simulating different scenarios and outputting data.

About the Project

Data Collection Process

1. Initialization

The robot begins its process after verifying that the wand and printer are properly connected. Users input parameters such as magnet type, orientation, and scan shape (e.g., square or hemisphere). For a square, the user specifies scan ranges in x, y, and z directions, along with the number of points. For a hemisphere, parameters include radius ranges, shell counts, and point distributions based on the Fibonacci sphere.

2. Scan Execution

The robot calculates and generates coordinates using Pythonâ€™s numpy library for square scans or the Fibonacci sphere method for hemispheres. Once the scan points are set, the robot executes G-code commands to move through each coordinate. At each point, the robot collects 100 magnetic field measurements, ensuring robust data accuracy.

3. Data Saving

The collected measurements are saved into a CSV file. This file includes detailed data for each scan point, such as x, y, and z coordinates, magnetic field values, and associated metadata.

4. Automatic Upload

Upon completing the scan, the CSV file is automatically uploaded to the designated Google Drive folder. Alongside the CSV, a JSON file is also uploaded. This file contains metadata about the scan, such as magnet type, orientation, wand details, and additional scan-specific information.

5. Data Accessibility

All uploaded files are organized by year and scan type, allowing for easy retrieval and analysis. The data can also be analyzed using a Python script, which computes error values for x, y, z, and distance from the scan data.

Project Contributions

Software updates were tracked using Git repositories on BitBucket. Each update was meticulously documented in a version history file, listing changes, commit IDs, and explanations for each feature.

New mechanical components were designed in Fusion 360, including updated marker holders and wand attachments. These components were optimized for improved stability, precision, and usability.
